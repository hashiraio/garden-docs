The **Garden Swap Skill** is a Claude Code skill that handles the full swap lifecycle ‚Äî from quoting to confirmation ‚Äî by responding to a natural-language swap request. It connects to the [Garden MCP server](/developers/ai/mcp), derives wallet addresses from your private key, executes the on-chain transactions, and polls for completion.

One sentence is enough to start a swap:

> *"Swap 0.001 BTC to wBTC"*
> *"Bridge my wBTC to Bitcoin"*
> *"Swap 0.5 SOL to BTC on Garden"*

---

## Compatibility

The Garden Swap Skill's **auto-loading** is Claude Code-specific, but the underlying signing scripts and credential helper (`keychain.ts`) work on **macOS, Linux, and Windows**. Any terminal-based AI tool that supports MCP and shell command execution can replicate the full automated swap flow with a small amount of manual setup.

| Tool | Automated swap flow | How |
|---|---|---|
| **Claude Code** | Native | Skill auto-loads; cross-platform `keychain.ts` manages the private key |
| **Gemini CLI** | Yes, with manual setup | MCP + Bun scripts + `keychain.ts run` |
| **Codex CLI** | Yes, with manual setup | MCP + Bun scripts + `keychain.ts run` |
| Claude Desktop | MCP only | No shell execution; sign via external wallet |
| Cursor / Windsurf | MCP only | No shell execution in chat; sign via external wallet |
| ChatGPT Desktop | MCP only | No shell execution; sign via external wallet |

| Platform | Keystore used |
|---|---|
| macOS | macOS Keychain (`security`) |
| Linux | GNOME Keyring / libsecret (`secret-tool`) ‚Äî requires `libsecret-tools` |
| Windows | Windows Credential Manager (PowerShell `PasswordVault`) |

## Using on Gemini CLI

Gemini CLI supports MCP servers and can execute shell commands, so the full quote ‚Üí sign ‚Üí initiate ‚Üí poll flow works. The difference from Claude Code is that you provide the private key as an environment variable instead of Keychain, and you load the skill instructions manually at the start of the session.

**Prerequisites**

- [Gemini CLI](https://github.com/google-gemini/gemini-cli) installed and authenticated
- [Bun](https://bun.sh) runtime ‚Äî `curl -fsSL https://bun.sh/install | bash`
- Garden MCP server connected (see [Connect your AI client ‚Üí Gemini CLI](/developers/ai/mcp#connect-your-ai-client))

<Steps>
  <Step title="Clone and install the signing scripts">
    ```bash
    git clone https://github.com/gardenfi/garden-agent.git
    cd garden-agent/garden-skill && bun install
    ```
  </Step>
  <Step title="Store your private key in your OS keystore">
    Run the setup script once ‚Äî it captures your key with masked input and saves it to your OS keystore (macOS Keychain, Linux libsecret, or Windows Credential Manager). The key never appears in the conversation:

    **macOS / Linux:**
    ```bash
    bash ~/garden-agent/garden-skill/scripts/setup-credentials.sh
    ```

    **Windows:**
    ```powershell
    powershell -ExecutionPolicy Bypass -File ~/garden-agent/garden-skill/scripts/setup-credentials.ps1
    ```

    Gemini retrieves the key at swap time via the cross-platform helper, which injects it into the subprocess env without ever printing it:

    ```bash
    bun run ~/garden-agent/garden-skill/scripts/keychain.ts run -- scripts/get-address.ts
    ```
  </Step>
  <Step title="Start a session with the skill instructions">
    Pass the `SKILL.md` as a system prompt so Gemini follows the same swap flow:

    ```bash
    gemini --system-prompt "$(cat ~/garden-agent/garden-skill/SKILL.md)"
    ```

    If your version of Gemini CLI uses a different flag (e.g., `-s`), consult `gemini --help`.
  </Step>
  <Step title="Describe your swap">
    ```
    swap 0.001 wBTC to BTC
    swap 0.5 SOL to BTC on Garden
    bridge USDC on Arbitrum to Bitcoin
    ```

    Gemini will call the Garden MCP tools for quotes and orders, then execute the Bun signing scripts via `keychain.ts run` to initiate on-chain.
  </Step>
</Steps>

## Usage

Describe your swap in plain language inside any Claude Code conversation ‚Äî no slash command needed:

```
swap 0.001 wBTC to BTC
bridge USDC on Arbitrum to Bitcoin
swap 0.5 SOL to strkBTC
swap 0.001 BTC to cbBTC on Base
convert 10 USDT to BTC
swap wBTC to BTC on testnet
```

**First-time setup (runs once)**

When you trigger a swap for the first time, the skill will:

1. **Register the Garden MCP servers** if they aren't already connected, then ask you to restart Claude Code.
2. **Run the credential setup script** to capture your private key ‚Äî it is stored directly in your OS keystore (macOS Keychain / Linux libsecret / Windows Credential Manager) and never appears in the conversation.

After the one-time setup, subsequent swaps only ask for chain-specific RPC URLs (saved to `config.json` for reuse).

## Supported chains

The skill works for any pair Garden has liquidity for ‚Äî always run `get_quote` to confirm before promising a swap.

| Family | Chains |
|---|---|
| **Bitcoin** | mainnet, testnet |
| **EVM** | Ethereum, Arbitrum, Base, BNB Chain, Botanix, Citrea, HyperEVM, MegaETH, Monad ‚Äî plus their testnets (Sepolia variants, Alpen, Arc, HyperCore) |
| **Starknet** | mainnet, Sepolia |
| **Solana** | mainnet, testnet |

Example pairs that work today: BTC ‚Üî wBTC, wBTC (Ethereum) ‚Üí cbBTC (Solana), USDC (Arbitrum) ‚Üí BTC, SOL ‚Üí strkBTC, wBTC (Starknet) ‚Üí SOL, USDC (Solana) ‚Üí wBTC (Ethereum), and many more.

All chains derive addresses from a single private key:

| Chain | Derivation |
|---|---|
| EVM (any chain) | secp256k1 ‚Üí Ethereum-format address (same across all EVMs) |
| Bitcoin | secp256k1 ‚Üí P2TR taproot address (network-specific) |
| Solana | 32-byte key as ed25519 seed ‚Üí base58 pubkey |
| Starknet | Key ground to STARK curve ‚Üí OpenZeppelin account address |

## How it works

When you describe a swap, the skill follows this sequence:

<Steps>
  <Step title="Verify MCP connectivity">
    Checks that `mcp__garden-mainnet__get_quote` is available. If not, registers the MCP servers and prompts you to restart Claude Code.
  </Step>
  <Step title="Collect credentials">
    Reads saved RPC URLs and addresses from `~/.claude/skills/garden/config.json`. Confirms the private key is present in your OS keystore via `keychain.ts exists` (the value itself is never read into the conversation). Only asks for missing values.
  </Step>
  <Step title="Get a quote">
    Calls `get_quote` with your source asset, destination asset, and amount. Displays the expected receive amount and solver.
  </Step>
  <Step title="Confirm">
    Shows the quote summary and waits for your approval before proceeding.
  </Step>
  <Step title="Create the order">
    Calls `create_order` with the quoted amounts. Receives an `order_id` and deposit instructions.
  </Step>
  <Step title="Execute on-chain">
    Sends the source-chain transaction. The exact method depends on the source chain:
    - **EVM source** ‚Äî polls `get_order_status` until a solver matches the order (to obtain `redeemer`, `timelock`, `secret_hash`), then builds both the ERC-20 `approve(htlc, amount)` and HTLC `initiate(redeemer, timelock, amount, secretHash)` calldata **locally** using viem's ABI encoding before broadcasting. The Garden hosted API does not return EVM calldata, so the client constructs it. Native-gas HTLC schemas (e.g. Botanix BTC, Citrea cBTC) skip the approve and send `initiate` as `payable` with `value = amount`.
    - **Bitcoin source** ‚Äî sends BTC to the HTLC deposit address via the bundled Bitcoin script
    - **Solana source** ‚Äî signs and broadcasts a versioned transaction
    - **Starknet source** ‚Äî auto-deploys the OZ account if needed, then broadcasts approval + initiate
  </Step>
  <Step title="Poll for completion">
    Calls `get_order_status` periodically and reports progress until `destination_swap.redeem_tx_hash` is set.
  </Step>
</Steps>

## Credential management

Sensitive values are kept out of the conversation entirely:

| Value | Storage |
|---|---|
| Private key | OS keystore via `scripts/keychain.ts` (macOS Keychain, Linux libsecret, or Windows Credential Manager) |
| RPC URLs (per EVM chain, Starknet, Solana) | `~/.claude/skills/garden/config.json` |
| Derived addresses (cached) | `~/.claude/skills/garden/config.json` |
| Bitcoin destination address | `~/.claude/skills/garden/config.json` |

The skill never prints your private key. `keychain.ts` exposes three commands:
- `exists` ‚Äî returns `"1"` or `"0"`, never the value
- `save <value>` ‚Äî writes to the OS keystore
- `run -- <script>` ‚Äî fetches the key, injects it into the subprocess env as `GARDEN_PRIVATE_KEY`, runs the script, exits. The key is scoped to that single subprocess only.

## Completion timeouts

The skill stops polling and reports the `order_id` for manual tracking if these timeouts are reached:

| Source chain | Timeout |
|---|---|
| EVM (any chain) | 5 minutes |
| Bitcoin | 60 minutes |
| Solana | 5 minutes |
| Starknet | 5 minutes |

Bitcoin confirmations on mainnet typically take 10‚Äì20 minutes. Testnet blocks are mined very slowly ‚Äî a BTC-source testnet order can take up to an hour. This is normal.
