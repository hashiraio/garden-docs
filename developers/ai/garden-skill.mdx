The **Garden Swap Skill** is a Claude Code skill that handles the full swap lifecycle, from quoting to confirmation, by responding to a natural-language swap request. It connects to the [Garden MCP server](/developers/ai/mcp), derives wallet addresses from your private key, executes the on-chain transactions, and polls for completion.

One sentence is enough to start a swap:

```
Swap 0.001 BTC to wBTC
Bridge my wBTC to Bitcoin
Swap 0.5 SOL to BTC on Garden
```

---

## Compatibility

Claude Code loads this skill automatically. Other terminal-based AI tools that support MCP and shell commands, like Gemini CLI and Codex CLI, can run the same swap flow with a bit of manual setup. The credential helper (`keychain.ts`) works cross-platform, on macOS, Linux, and Windows.

| Tool | Automated swap flow | How |
|---|---|---|
| Claude Code | Native | Skill auto-loads; cross-platform `keychain.ts` manages the private key |
| Gemini CLI | Yes, with manual setup | MCP + Bun scripts + `keychain.ts run` |
| Codex CLI | Quote and order creation are automated; signing is manual | MCP for quotes/orders; hands you the exact `keychain.ts run` command to execute yourself |
| Claude Desktop | MCP only | No shell execution; sign via external wallet |
| Cursor / Windsurf | MCP only | No shell execution in chat; sign via external wallet |
| ChatGPT Desktop | MCP only | No shell execution; sign via external wallet |

| Platform | Keystore used |
|---|---|
| macOS | macOS Keychain (`security`) |
| Linux | GNOME Keyring / libsecret via `secret-tool` (requires `libsecret-tools`) |
| Windows | Windows Credential Manager (PowerShell `PasswordVault`) |

## Using on Codex CLI

Codex CLI runs in an isolated shell that can't trigger your OS keystore's interactive unlock prompt, so unlike Claude Code, it never runs the signing/initiate command itself. Credential setup and MCP configuration work exactly the same as everywhere else, and the only difference is that Codex hands you the exact signing command to run yourself, in your own terminal, instead of running it directly. You also configure the MCP servers and load the skill instructions manually at the start of the session, since Codex CLI doesn't auto-register or auto-load either.

**Prerequisites**

- [Codex CLI](https://github.com/openai/codex) installed and authenticated
- [Bun](https://bun.sh) runtime: `curl -fsSL https://bun.sh/install | bash`
- Garden MCP server connected (see [Connect your AI client → OpenAI Codex CLI](/developers/ai/mcp#connect-your-ai-client))

<Steps>
  <Step title="Clone and install the signing scripts">
    ```bash
    git clone https://github.com/gardenfi/garden-skill.git
    cd garden-skill && bun install
    ```
  </Step>
  <Step title="Store your private key in your OS keystore">
    Run the same setup script used by Claude Code:

    **macOS / Linux:**
    ```bash
    bash ~/garden-skill/scripts/setup-credentials.sh
    ```

    **Windows:**
    ```powershell
    powershell -ExecutionPolicy Bypass -File ~/garden-skill/scripts/setup-credentials.ps1
    ```
  </Step>
  <Step title="Start a session with the skill instructions">
    ```bash
    codex --instructions "$(cat ~/garden-skill/SKILL.md)"
    ```

    Check `codex --help` if the flag name differs in your version.
  </Step>
  <Step title="Describe your swap">
    ```
    swap 0.001 BTC to wBTC
    bridge my wBTC to Bitcoin on mainnet
    ```

    Codex calls the Garden MCP tools to get a quote and create the order, then gives you the exact command to run yourself. Codex never inspects your private key or runs signing commands directly:

    ```bash
    bun run ~/garden-skill/scripts/keychain.ts run -- scripts/get-address.ts
    ```

    Run it in your terminal, tell Codex when it's done, and it polls the order and reports the transaction hashes.
  </Step>
</Steps>

## Installation

**Prerequisites**

- [Claude Code](https://claude.ai/code) installed and authenticated
- [Bun](https://bun.sh) runtime: `curl -fsSL https://bun.sh/install | bash`
- **Linux only:** `libsecret-tools`: `sudo apt install libsecret-tools` or `sudo dnf install libsecret`

**Steps**

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/gardenfi/garden-skill.git
    cd garden-skill
    ```
  </Step>
  <Step title="Copy the skill to Claude Code's skills directory">
    The directory name must be exactly `garden`:

    ```bash
    cp -r . ~/.claude/skills/garden
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    cd ~/.claude/skills/garden && bun install
    ```
  </Step>
  <Step title="Restart Claude Code">
    Quit and reopen Claude Code. The skill is now active.
  </Step>
</Steps>

**Updating the skill**

```bash
cd garden-skill
git pull
cp -r . ~/.claude/skills/garden
cd ~/.claude/skills/garden && bun install
```

## Usage

Describe your swap in plain language inside any Claude Code conversation. No slash command needed:

```
swap 0.001 wBTC to BTC
bridge USDC on Arbitrum to Bitcoin
swap 0.5 SOL to strkBTC
swap 0.001 BTC to cbBTC on Base
convert 10 USDT to BTC
swap wBTC to BTC on testnet
```

**First-time setup (runs once)**

When you trigger a swap for the first time, the skill will:

1. **Register the Garden MCP servers** if they aren't already connected, then ask you to restart Claude Code.
2. **Run the credential setup script** to capture your private key. It is stored directly in your OS keystore (macOS Keychain / Linux libsecret / Windows Credential Manager) and never appears in the conversation.

After the one-time setup, subsequent swaps only ask for chain-specific RPC URLs (saved to `config.json` for reuse).

## Supported chains

The skill works for any pair Garden has liquidity for. Always run `get_quote` to confirm before promising a swap.

| Family | Chains |
|---|---|
| **Bitcoin** | Mainnet, testnet |
| **EVM** | Ethereum, Arbitrum, Base |
| **Starknet** | Mainnet, Sepolia |
| **Solana** | Mainnet, testnet |

For the complete list, call `get_chains` on the Garden MCP server.

Example pairs that work today: BTC ↔ wBTC, wBTC (Ethereum) → cbBTC (Solana), USDC (Arbitrum) → BTC, SOL → strkBTC, wBTC (Starknet) → SOL, USDC (Solana) → wBTC (Ethereum), and many more.

All chains derive addresses from a single private key:

| Chain | Derivation |
|---|---|
| EVM (any chain) | secp256k1 → Ethereum-format address (same across all EVMs) |
| Bitcoin | secp256k1 → P2TR taproot address (network-specific) |
| Solana | 32-byte key as ed25519 seed → base58 pubkey |
| Starknet | Key ground to STARK curve → OpenZeppelin account address |

## How it works

When you describe a swap, the skill follows this sequence:

<Steps>
  <Step title="Verify MCP connectivity">
    Checks that `mcp__garden-mainnet__get_quote` is available. If not, registers the MCP servers and prompts you to restart Claude Code.
  </Step>
  <Step title="Collect credentials">
    Reads saved RPC URLs and addresses from `~/.claude/skills/garden/config.json`. Confirms the private key is present in your OS keystore via `keychain.ts exists` (the value itself is never read into the conversation). Only asks for missing values.
  </Step>
  <Step title="Get a quote">
    Calls `get_quote` with your source asset, destination asset, and amount. Displays the expected receive amount and solver.
  </Step>
  <Step title="Confirm">
    Shows the quote summary and waits for your approval before proceeding.
  </Step>
  <Step title="Create the order">
    Calls `create_order` with the quoted amounts. Receives an `order_id` and deposit instructions.
  </Step>
  <Step title="Execute on-chain">
    Sends the source-chain transaction. The exact method depends on the source chain:
    - **EVM source**: polls `get_order_status` until a solver matches the order (to obtain `redeemer`, `timelock`, `secret_hash`), then builds both the ERC-20 `approve(htlc, amount)` and HTLC `initiate(redeemer, timelock, amount, secretHash)` calldata **locally** using viem's ABI encoding before broadcasting. The Garden hosted API does not return EVM calldata, so the client constructs it. Native-gas HTLC schemas (e.g. Botanix BTC, Citrea cBTC) skip the approve and send `initiate` as `payable` with `value = amount`.
    - **Bitcoin source**: sends BTC to the HTLC deposit address via the bundled Bitcoin script
    - **Solana source**: signs and broadcasts a versioned transaction
    - **Starknet source**: auto-deploys the OZ account if needed, then broadcasts approval + initiate
  </Step>
  <Step title="Poll for completion">
    Calls `get_order_status` periodically and reports progress until `destination_swap.redeem_tx_hash` is set.
  </Step>
</Steps>

## Credential management

Sensitive values are kept out of the conversation entirely:

| Value | Storage |
|---|---|
| Private key | OS keystore via `scripts/keychain.ts` (macOS Keychain, Linux libsecret, or Windows Credential Manager) |
| RPC URLs (per EVM chain, Starknet, Solana) | `~/.claude/skills/garden/config.json` |
| Derived addresses (cached) | `~/.claude/skills/garden/config.json` |
| Bitcoin destination address | `~/.claude/skills/garden/config.json` |

The skill never prints your private key. `keychain.ts` exposes three commands:
- `exists`: returns `"1"` or `"0"`, never the value
- `save <value>`: writes to the OS keystore
- `run -- <script>`: fetches the key, injects it into the subprocess env as `GARDEN_PRIVATE_KEY`, runs the script, exits. The key is scoped to that single subprocess only.

## Completion timeouts

The skill stops polling and reports the `order_id` for manual tracking if these timeouts are reached:

| Source chain | Timeout |
|---|---|
| EVM (any chain) | 5 minutes |
| Bitcoin | 60 minutes |
| Solana | 5 minutes |
| Starknet | 5 minutes |

Bitcoin confirmations on mainnet typically take 10-20 minutes. Testnet blocks are mined very slowly, and a BTC-source testnet order can take up to an hour. This is normal.
